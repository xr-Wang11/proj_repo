# 模块 12：网页 UI 初版

模块 12 负责把前面已经完成的核心能力接到一个真实可打开的 HTML 页面上。

## 当前实现位置

- `scripts/build-web-bundle.js`
- `web/app-entry.js`
- `web/index.html`
- `tests/web/bundle-build.test.js`

## 目标

模块 12 的目标不是引入前端框架，而是先交付一个：

- 可直接打开
- 可接收输入
- 可调用核心计算链路
- 可展示结果文本和 LaTeX 字符串

的网页版本初版。

## 当前结构

### `scripts/build-web-bundle.js`

职责：

- 扫描 `web/app-entry.js` 及其相对依赖
- 生成浏览器可执行的单文件 bundle
- 输出到 `web/app.bundle.js`

当前只支持项目内相对 `require()`。

### `web/app-entry.js`

职责：

- 挂载页面交互
- 使用模块 11 的网页输入协议
- 监听键盘、按钮、下拉框
- 驱动页面渲染

### `web/index.html`

职责：

- 提供真实页面结构和样式
- 展示输入区、按钮区、结果区、错误区

## 当前页面能力

已经支持：

- 表达式键盘输入
- 运算符/函数按钮插入
- 光标移动
- 删除与清空
- 结果文本显示
- 表达式 LaTeX 字符串显示
- 结果 LaTeX 字符串显示
- 解析错误显示
- 求值错误显示
- `displayMode` 切换
- `angleUnit` 切换

## 当前输入规则

网页层完全复用模块 11：

- 数字、字母、小数点可键盘输入
- 运算符和函数必须点击按钮

这保证：

- 后续网页 UI 不会和 CLI 或核心引擎分叉出不同的表达式协议

## 当前页面风格

初版界面采用了偏“仪表盘 / 工程台”方向的视觉：

- 暖色纸面背景
- 半透明卡片
- 明确的按钮分组
- 结果区和错误区分开

目标是先建立有辨识度的网页容器，而不是默认表单页。

## 当前限制

- LaTeX 目前仍然显示为字符串，尚未接入 KaTeX / MathJax
- 没有用户函数面板
- 没有历史记录面板
- 没有持久化设置

这些限制不影响它作为第一版网页 UI 的作用。

## 如何构建

生成 bundle：

```powershell
node scripts\build-web-bundle.js
```

然后直接打开：

```text
web/index.html
```
